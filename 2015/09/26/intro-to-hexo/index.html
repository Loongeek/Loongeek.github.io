概述

Hexo是个简单有效的博客框架，主要使用Markdown语言，能够快速生成静态文件。

安装

  • 安装Node.js
    先安装nvm(Node Version Manager):

    $ curl https://raw.github.com/creationix/nvm/master/install.sh | sh

    再安装Node.js: 

    $ nvm install 0.12

  • 安装Git
    Win: 直接下载安装git
    Mac: 用Homebrew安装
    Ubuntu: sudo apt-get install git-core

  • 用npm(Node Package Manager)安装Hexo

    $ npm install -g hexo-cli

    Mac用户在编译时可能遇到问题。首先安装Xcode，然后打开Xcode，选择Preferences->Download->Command Line Tools->Install安装命令行工具

    配置

  • 在指定目录下初始化Hexo

    $ hexo init <folder>
$ cd <folder>
$ npm install

    初始化完成后的目录结构

      .
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
├── public
|   ├── _drafts
|   └── _posts
└── themes

    _config.yml:站点配置文件
    package.json:应用程序数据
    scaffolds:文章模板
    scripts:JS,会自动执行
    source:站点内容
    public:生成的静态页面
    theme:静态站点主题风格

    • 全局配置
      配置全局的_config.yml
      Site

      *title* 站点标题
*subtitle* 站点副标题
*description* 站点描述(搜索关键字，空格分隔)
*author* 作者
*language* 站点语言
*timezone* 时区

      URL

      *url* 站点url
*root* 站点根目录
*permalink* 永久链接(静态页面链接的生成格式)
*permalink_default* 永久链接默认值

      站点在子目录下的，将路径改为子目录(如http://wesite/subblog)，同时将root改为子目录(/subblog/)
      Directory 修改各部分的默认目录
      Writing

      *new_post_name* 新博客的文件名格式(默认为:title.md)
*default_layout* 默认布局
*titlecase* 标题转换大小写??
*external_link*    在新窗口打开外链
*filename_case* 文件名转换到小写(1)或大写(2)
*render_drafts* 显示草稿？
*post_asset_folder* 使能Asset文件夹
*relative_link* 生成相对root的相对链接?
*future* 显示未来博客??
*highlight* 代码块设置??

      Category & Tag

      *default_category* 默认分类
*category_map* 目录地图??
*tag_map* 标签地图??

      Date/Time format

      *date_format* 日期格式(默认为MMM D YYYY)
*time_format* 时间格式(默认为H:mm:ss)

      Pagination

      *per_page* 一页显示多少博客(0关闭分页)
*pagination_dir* 分页目录??

      Extensions

      *theme* 使用主题名称
*deploy* 部署设置

      命令

      init

      $ hexo init [folder]

      在指定目录初始化站点

      new

      $ hexo new [layout] <title>

      创建新博文，标题中可使用空格，但需要加双引号

      generate

      $ hexo generate

      生成静态页面，-d 生成后部署, -w查看文件变化

      publish

      $ hexo publish [layout] <filename>

      将草稿变成发布状态

      server

      $ hexo server

      运行本地服务器，默认路径http://localhost:4000/

      deploy

      $ hexo deploy

      部署网站，-g在部署前先生成

      render

      $ hexo render <file1> [file2] ...

      渲染文件 -o输出目标

      migrate

      $ hexo migrate <type>

      合并其它博客系统的内容

      clean

      $ hexo clean

      清理中间文件和生成的文件

      list

      $ hexo list <type>

      列出所有路由

      version

      $ hexo version

      版本信息

使用方法

发表

创建新文章

$ hexo new [layout] <title>

Layout
post 在source/_posts下，发布的文章
page 在source下，单独页面形式的文章
draft 在source/_drafts下，草稿

Filename
Hexo默认使用文章标题做为文件名。可以在_config.yml的new_post_name中修改，具体占位符的使用方法略

Drafts
草稿不会生成到静态页面，可以通过publish命令发布

Scaffolds
使用 hexo new XXX "aaaa"命令时，Hexo会从Scaffolds目录下找寻名为XXX.md的文件，做为新生成页面的模板。

前页

YAML或者JSON形式的页面配置。YAML形式如下：

title: XXXX
date: xxxxx
---

冒号必须有空格，以—-结束

  • layout 指定布局
  • title 指定标题
  • date 指定发布时间
  • updated 指定更新时间
  • comments 使能评论功能
  • permalink 覆盖默认的永久链接
  • tags 标签(pages类型不可用)

  • categories 目录，与标签相比是有层次结构的(pages类型不可用)

    categories:
- Sports
- Baseball
tags:
- Injury
- Fight
