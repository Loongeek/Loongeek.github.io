Swing的刷新方法一览

Swing的刷新方法

  • Paint()
    可以用paintComponent()代替paint()。paint()会依次调用paintBorder(), paintComponent()和paintChildren()。一般不要直接调用该方法，应当调用repaint()

  • repaint()
    不能被覆盖。控制着update() -> paint()的消息循环。当你变更了组件的外观(除了更改大小)，就应当调用该方法

  • validate()
    告诉组件刷新自己。如果你更改了组件大小，或者变更了它的子组件(添加、删除、改变大小)，应当调用该方法。也许调用revalidate()更好

  • update()
    负责清除组件并调用paint()。一般不要直接调用它，应当调用repaint()。如果在做动画时需要快速刷新，可以覆盖该方法让它直接调用paint()

  • updateUI()
    如果你在组件显示后，又更改了UI的风格，应当调用该方法

  • invalidate()
    标记组件是失效的，意味着组件必需被刷新。但是这只是个标记，可能会多个组件在之后适当的时间一起刷新

  • validate()
    重新布局。意味着布局管理器会重新设置失效的组件及其子组件的大小

  • revalidate()
    相当于invalidate()+validate()。标记组件失效并处理

Hexo发布到Github

  1. 在Github上创建一个带readme的新repository

  2. 修改hexo主目录的_config.yml，添加

    deployer:
    type: git
    repository: <github的repository路径>
    branch: master

  3. 如果报错说找不到deployer,就执行以下代码安装：

    npm install hexo-deployer-git --save

  4. 安装完成后执行

    hexo deploy

ssh配置

ssh与scp

基本用法:

ssh user@hostname command
scp [-r] [user@host1:]file1 user@host2:file2

ssh免密配置

  1. 本机运行

    ssh-keygen -t rsa

    在~/.ssh/下生成了私钥id_rsa和公钥id_rsa.pub

  2. 利用scp将id_rsa.pub传送到远程服务器的用户目录~/.ssh/下，
    改名为authorized_keys，如果已有这个文件，就append到该文件
  3. 修改权限：
    • 本地的.ssh权限改为644, id_rsa权限改为600,known_hots权限改为644
    • 远程的.ssh权限改为644，authorized_keys权限改为644
