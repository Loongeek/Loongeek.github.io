汇总术语。

A

B

C

Closure

闭包

  • 与数学上的闭包不太一样，指的是包含了自己上下文的代码结构
  • 通常以函数变量或者lambda表达式的方式存在
  • 能够访问自己上下文的内容，即使自己已经不在上下文中了。

CORS

Cross-Origin Resource Sharing， 跨域资源共享
CORS是一种允许当前域（domain）的资源（比如html/js/web service）被其他域（domain）的脚本请求访问的机制，通常由于同域安全策略（the same-origin security policy）浏览器会禁止这种跨域请求。

CSI

Client Side Includes，客户端包含，动态页面缓存技术的一种

  • 概念：整个页面做为静态html页面，在需要动态的地方则通过iframe,javascript或ajax来动态加载！
  • 优点：相对比较简单，不需要服务器端做改变和配置；
  • 缺点：不利于搜索引擎优化(iframe方式), javascript兼容性问题，以及客户端缓存问题可能导致更新后不能及时生效！对于客户体验也不够好！

D

Daemon

守护进程

  • 在类UNIX系统，以d结尾的进程，不依赖于任何终端。
  • 通常用来执行后台进程
  • fork->结束父进程->setsid 改会话进程组->改变工作目录到根目录->umask 改变屏蔽权限->通过发signal来kill

DDos

Distributed Denial of Servic，攻击分布式拒绝服务
分布式地向目标发送大流量的正常或者伪装成正常的请求，导致对方无法处理正常的请求

DRY

Don’t Repeat Yourself

Dummy Object

哑对象作为属性或者参数使用，用于待测试内容需要该对象做为参数，但不真正使用对象的情况（如记数，或者测试打印字串速度，但不关心字串的实际内容等等）

E

ESI

Edge Side Includes，边界包含，动态页面缓存技术的一种

  • 概念：通过使用简单的标记语言来对那些可以加速和不能加速的网页中的内容片断进行描述，每个网页都被划分成不同的小部分分别赋予不同的缓存控制策略，使Cache服务器可以根据这些策略在将完整的网页发送给用户之前将不同的小部分动态地组合在一起。
  • 优点：通过这种控制，可以有效地减少从服务器抓取整个页面的次数，而只用从原服务器中提取少量的不能缓存的片断，因此可以有效降低原服务器的负载，同时提高用户访问的响应时间。
    与SSI不同的是，ESI多在缓存服务器或代理服务器上执行！可用于缓存整个页面或页面片段，比较适合用于缓存服务器上；
  • 缺点：目前支持ESI的软件还比较少，官方更新也略显缓慢，因此使用不是很广！

F

Fake Object

假对象，使用比真实对象更简单的实现（如在测试中使用内存数据库，而不进行真正的数据库访问)

K

L

LDAP

Lightweight Directory Access Protocol，轻量级目录管理
分布式访问，易查询不易写入，树状结构。通常用来存一些不太会更改的内容，比如注册用户，电话号码等等

M

Mock Object

Mock对象,验证测试代码的间接输入。但是输出是在测试运行前预先定义好的。与Stub相比，Mock可以定义相对复杂的交互，并且要对期望进行验证。

O

ORM

Oject Relationship Mapping, 对象关系映射
将数据库映射到对象上，数据库的表就是类，列就是类的成员变量，每一行是类实例；数据库的关系用成员对象或者成员对象数组表示？？

R

REST

Representational State Transfer, 表述性状态传输

  • URI Unique Resource ID 独立资源标识符
  • URL ⊂ URI
  • 原则：
    • URI名称中无动词
    • 动作用GET POST PUT DELETE表示
    • 不能在URI中加版本号（版本号放在Accept中）

S

SSI

Server Side Includes，服务端包含，动态页面缓存技术的一种

  • 概念：通过注释行SSI命令加载不同模块，构建为html，实现整个网站的内容更新；需要服务器模块支持，比如：apache服务器需要开启mod_include模块
  • 优点：不受具体语言限制，比较通用，只需要Web服务器或应用服务器支持即可，Ngnix、Apache、IIS等对此都有较好的支持。
  • 缺点：SSI只能在当前服务器上包含加载，不能够直接包含其他服务器上的文件，即不能跨域包含；

T

Test Double

测试替身
做单元测试时为了隔离外部关联而创建的假的外部测试对象或者环境。
当需要网络访问，数据库访问，文件系统访问，不能独立运行，需要额外配置时，就认为不是单元测试；这些情况下就需要测试替身来模拟相应的外部环境。
测试替身包括Stub, Mock对象, Spy, Fake对象,Dummy对象

Test Spy

验证测试代码的间接输出，在测试完成前不定义期望值，而是在测试完成后检查期望值。可以保证四步法测试各步骤的分离（setup->exercise->verify->teardown）未细看
test spy

Test stub

给测试代码提供“间接性输入”,通常是硬编码的输入-输出对。与mock不同，stub只对输入进行输出响应，不判断期望的结果。

X

XSS

Cross Site Script Execution,跨网站攻击
向一个用户发送另一个用户输入的恶意内容
