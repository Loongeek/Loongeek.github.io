模型

  • 结构 + 操作 + 约束
  • 列是属性，行是元组
  • 模式 = 表的属性集合(属性不能有重复项)
  • 关系 = 某模式下元组的集合(元组不能有重复项)
  • 键 = 具有唯一性的属性组合（对所有可能实例）

模型与SQL

  • 关系 = SQL table
  • 某种子模式下的关系 = SQL view
  • SQL数据类型：
    CHAR VARCHAR BIT BIT VARING BOOLEAN(UNKNOWN) INT INTEGER FLOAT REAL DOUBLE PRECISION DATE TIME
  • SQL操作：
    • CREATE TABLE name(prop1 type(xxx), prop2 type(xxx), …);
    • DROP TABLE name;
    • ALTER TABLE name ADD prop type(xxx);
    • ALTER TABLE name DROP propname;
    • 建议用NULL代替真实值
    • propname CHAR(1) DEFAULT ‘?’
    • CREATE TABLE name(…, propX type(xxx) PRIMARY KEY, propY type(xxx) UNIQUE, … ); //主键不能有NULL, UNIQUE键可以有NULL
    • CREATE TABLE name(…, PRIMARY KEY(xxx,xxx,xxx), …); //复合主键

设计理论

  • FD函数依赖： A1,A2,…. -> B1,B2,…. (对所有可能实例成立)
  • 键：能决定其它所有属性的最小集合
  • 超键：含键的属性集
  • 分解/组合规则
  • 平凡FD: Ax -> Bx, 若Bx⊆Ax。平凡依赖规则
  • 闭包：{A1,A2,….An}⁺, 其中A1,A2,…An是属性集
  • 闭包算法：逐步添加法
  • 传递规则：可消去平凡FD（自反律，增广律）
  • 超键的闭包是所有属性
  • 基本集，最小化基本集
    • 右边是单一属性
    • 删掉任一FD,就不是基本集了
    • 任一FD,左边删掉任一属性，就不是基本集了

模式设计

  • 异常的3种形式：分解
  • BCNF：R 非平凡FD左边是超键
    二元关系都属于BCNF
    简单理解为：如果键的部分属性集合能决定其它某些属性，就单独拿出来做一张新表(新关系)
  • 分解法则：
    1. 不违例，返回
    2. 违例,X->Y; R1={X+} R2={!X+ ∪ X}
    3. 递归
  • chase检验：测无损
  • 下标替换法则
  • 反向证明
  • 3NF
    • 增加了 “Bs都是某键成员”
    • 非平凡FD，要么左边是超键，要么右边是主属性
  • 3NF分解算法：
    1. 找出F最小基本集G
    2. G中每个X->A, {XA}分解出来
    3. 若不含超键，增加关系模式为R的任一键
  • 最小基本集测试原则：3个
  • 无损连接：必有超键或键（后者是3NF的关键）
  • 依赖保持：按依赖分解
