Gem,Gemfile,Bundle

Gem命令

  • gem search (regex) [-d] 按名字查找gem,支持正则表达式，加-d显示详细信息
  • gem install xxx [—no-doc] 安装gem,可以选择是否支持文档
  • gem list, gem uninstall
  • ri xxx 查看已安装的gem文档
  • gem query —remote 查看远程机上有哪些gem
  • gem fetch 获取gem但不安装
  • gem unpack 解包

Gemfile

  • 第一行指定源 source "https://gems.ruby-china.org" (链接是国内镜像源)
  • 源的优先级：显式定义的源，路径或者git -> Gemfile源按先后顺序
  • 有些源要验证身份：
  • bundle config my_gem_source.com my_username:my_password 在bundle中设置，不会提交到版本管理中
  • source "https://username:password@my_gem_source.com" 在Gemfile中设置，会提交到版本管理中
  • 指定gem版本：gem "gem_name","operand 1.2"
    operand: = != > < >= <= ~>（主版本范围内的任意版本，例如"~> 2.5.0"相当于">= 2.5.0","< 2.6.0"）
  • gem "gem_name", require: false 指定gem是否由bundle来require
    gem "gem_name", require: [dir1,dir2,...] 自动require哪些目录
    gem "gem_name", :require => "filename" 指定require的主文件名。
    如果没有使用自动require，要在代码中手动require
  • gem "my_gem", group: :development 指定gem所在组，不指定的话，默认在:default组
    或者用block指定多个组，多个gem：

非optional的gem不想安装：bundle install --without development ...
optional的gem想安装：bundle install --with development ...

  • gem "my_gem", platform: [xxx,xxx] 指定平台，类似group，但是由bundle自己判断
  • Gemfile.lock记录了本机安装的gem版本，这样在其它环境下执行bundle install会安装同样的版本，而不会更新gem版本。最好将Gemfile.lock放入版本控制中

Bundle

  • 显示所有的依赖包 bundle show
  • 显示指定gem包的安装位置 bundle show [gemname]
  • 检查系统中缺少那些项目依赖的gem包 bundle check
  • 安装项目依赖的所有gem包 此命令会尝试更新系统中已存在的gem包 bundle install
  • 安装指定的gem包 bundle install [gemname]
  • 更新系统中存在的项目依赖包，并同时更新项目Gemfile.lock文件 bundle update
  • 更新系统中指定的gem包信息，并同时更新项目Gemfile.lock中指定的包信息 bundle update [gemname]
  • 使用bundle lock来锁定当前环境，这样便不能通过bundle update来更新依赖包的版本，保证了统一的环境 bundle lock
  • 解除锁定 bundle unlock
  • bundle package会把当前所有信赖的包都放到 ./vendor/cache/ 目录下，发布时可用来保证包版本的一致性 bundle package
