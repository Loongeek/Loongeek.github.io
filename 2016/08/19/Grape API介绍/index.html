Grape API介绍。参考 Grape API文档
Grape是REST-like的ruby的API框架。可以在Rack上运行，也可以做为DSL来补充已有的框架(Rails,Sinatra)

简介

+ 声明为`Grape::API`的子类，指定version, format, prefix等
+ 定义helpers
+ 定义resource,在resource中定义不同的响应

挂载

  • Rack: 在config.ru中写run Modulename::Classname
  • Sinatra: 在config.ru中写
    use Rack::Session::Cookie
    run Rack::Cascade.new [API, Web] //Cascade依次尝试处理并返回第一个不是404的结果
  • Rails:
    把API类文件放到app/api中，app/api/Modulename/Classname.rb
    修改application.rb:
    config.paths.add File.join('app', 'api'), glob: File.join('**', '*.rb')
    config.autoload_paths += Dir[Rails.root.join('app', 'api', '*')]
    修改config/routes:
    mount Modulename::Classname => '/'
    注意：如果使用4.0+的Rails并使用默认的ActiveRecord模型层,可能需要使用hashie-forbidden_attributesgem,它会禁止模型层的strong_params安全特性，这样就允许你使用Grape自己的参数验证来代替。

版本

  • version 'v1', using: :path，在URL中指定版本
  • version 'v1', using: :header, vendor: 'twitter', 在header中指定版本
  • version 'v1', using: :accept_version_header，用accept_version指定版本
  • version 'v1', using: :param, parameter: 'v'，用参数指定版本

描述

参数

  • declared,只会访问声明了的参数
  • 声明参数：params do requires :xxx, type :xxx do [requires :xxx,type,:xxx]end end
  • declared默认包含nil参数，可以设置include_missing: false禁止包含

参数验证

  • 使用params来做验证
  • optional形式的参数可以指定默认值，但是默认值应当包含在取值范围内
  • 支持的参数类型:
    Integer Float BigDecimal Numeric Date DateTime Time Boolean
    String Symbol Rack::Multipart::UploadedFile (alias File) JSON
  • 自定义参数验证：在类中定义parse(String)返回参数，会自动验证；或者指定参数时用coerce_with指定验证方法
  • 参数依赖关系：given :xxxx do end
  • 内建验证器：allow_blank, values, regexp
    mutually_exclusive，exactly_one_of, at_least_one_of, all_or_none_of(参数互斥，不同时出现)
  • 自定义验证器类(略)

  • 写回：header ‘xxxx’,’xxxx’
