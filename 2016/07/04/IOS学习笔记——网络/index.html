概念

包

包通常包括三个部分：头(header) 有效数据(payload) 尾(trailer)

TCP/IP网络模型

  • 链路层/物理层：
    常用的物理层：Wifi, 蜂窝网，以太网，蓝牙.
    通常由操作系统处理
    不同的链路层支持的包大小不一样，用 MTU(maximum transmission unit)表示
  • IP层：
    将包从主机发送到目标；维护路由；分包/组包。
    分包大小不合适可能会产生额外花费，操作系统通常采用链路MTU发现的技术来决定最大 的传输包
  • 传输层：
    • TCP(transmission control protocol,传输控制协议)。
      保证数据包按顺序、完整到达目标端
      拥塞控制和流控
      数据流，数据被看作字节序列，而不是像UDP那样的离散数据记录。如果要通过TCP发送离散数据记录，需要自己设置记录边界
      链路MTU发现
    • UDP(user datagram protocol,用户数据报协议)。
      不保证数据到达目标，由用户自己处理重传(低延时的场景，比如实时游戏等)
      IPV4下支持广播
      支持多播
      两者都在IP的基础上增加了端口的概念，支持在一个主机上执行多个网络服务
    • ICMP(Internet control message control),为TCP和UDP提供额外的连接异常检测功能，提供ping功能
    • TLS(transport layer security)：在传输层之上，提供传输层安全
    • L2TP和PPTP：在传输层之上，用于VPN协议
  • 应用层：HTTP和FTP等

延时

延时指的是一个包从发出，到目标，目标应答，回到主机的整个回环时间。
由传播速度，路由延迟，重传时间，信号传播延迟等决定

寻址和域名

  • 链路层寻址
    链路层用一个全局硬件ID来区分：
    Ethernet,Wifi —— MAC
    蓝牙 —— 蓝牙硬件ID
    蜂窝 —— IMEI/ESN/MEID
  • IP层寻址
    用IP地址来区分：
    IPv4 —— 4字节地址
    IPv6 —— 128bit(16字节)地址
  • DNS(Domain Name System，域名系统)
    比字节好记；IP改变不会影响基于域名的服务；可以通过多个IP访问同一域名；可以用多个服务器服务一个域名
    域名：www(主机).loongeek(域名).com(TLD,顶级域名)
    可以使用whois工具查看域名归属

包路由

  • LAN使用ARP来完成IP地址到MAC的转换
  • Internet使用路由器
    边界路由器处理本地路由或者连接到核心路由器；
    核心路由器连接到其它路由器，使用BGP或者RIP协议
  • IPv4: 网络号+主机号，子网掩码(可以用/n的形式表示，表示前n个bit是1，e.g：/8 = 255.0.0.0)
    广播地址：主机bit全为1
    网络地址：主机bit全为0(保留)
    路由器地址：通常为主机号=1
    特殊地址：环回 127.0.0.1 多播 224.0.0.0/4 本地 192.168.0.0/16
  • IPv6: 网络号(64bit):全局路由前缀+子网ID
    主机号(64bit)
    特殊地址： ::1/128环回 FF00::/8多播 FEC0::/10本地
  • 防火墙：阻止特定端口；阻止错包或者伪装包；包审查；NAT(network address translation,改变包的源和目标地址或端口)

动态地址分配

  • IPv4使用DHCP(Dynamic Host Configuration Protocol)
  • IPv6使用邻居发现，获取同一个物理网络其它主机的物理地址，代替ARP
    使用SLAAC(stateless address autoconfiguration),代替DHCP

要点

现实网络的编程要点

  • 有效用电和流量
    • 打包传输数据，尽可能保持Idle：一次下载足够资源，而不是分多次下载，每次下载一点点
    • 下载尽量小的资源，并且在本地缓存资源：保证下载的资源尽可能的小，并且合理使用本地Cache（参考 NSURLCache）
  • 得体的处理网络问题
    • 设计时考虑可变的网络接口
      始终尝试进行连接，不要猜测网络接口，也不要沿用之前的判定结果。
      连接失败时使用 SCNetworkReachability进行诊断，并自动连接，不要让用户参与。一般采用非模式的形式显示网络状态。
      后台执行时，不要提示用户，自动维持连接。使用 SCNetworkReachabilityAPI。失败时重试不要太快，逐渐拉长重试时间
    • 设计时考虑可变网速：网速只有使用中才能知道，软件根据网速来做一些处理，比如选用更小的资源
    • 设计时考虑高延时：如果数据没有关联性，可以并发地发出请求。 NSURLConnection可以使用 HTTP pipeline来自动实现并发（ NSMutableURLRequest.setHTTPShouldUsePipelining）

评估网络编程API

  • 网络层次表
层 流协议 URL 服务发现
Foundation NSStream NSURLConnection NSURLRequest NSNetService
Core Foundation CFStream CFHTTPMessage CFNetService
POSIX kqueue libcurl DNS
  • 常见网络编程方案
    • 点对点网络游戏：使用 GameKit，支持点对点多人游戏，支持语音通信
    • 点对点网络应用：IOS下使用 Multipeer Connectivity框架
    • 连接到网络服务器：使用HTTP/HTTPS
    • 其它情况：直接使用socket(POSIX或者Core Foundation层)或者socket流(Core Foundation层)；支持TLS（SSL的后继）

网络服务（略）

显示网页和多媒体服务

  • 在默认程序中打开网页内容或者流媒体：IOS下使用 UIApplication.openURL
    注：可以使用 mailto://发邮件或者 tel://打电话)
  • 在自己的应用中打开网页内容：IOS下使用 UIWebView.loadRequest。但是不提供对底层连接的访问，如果连接无法解析，方法会自动失败
  • 在自己的应用中显示流媒体：在IOS中使用 Media Player框架进行简单处理，或者使用 AV Foundation框架进行复杂处理

HTTP和HTTPS请求

  • 不同应用场景：
    • 后台消息推送： NKAssetDownload
    • 下载文件： NSURLDownload
    • 其它情况： NSURLConnection
  • 使用Foundation框架：
    • 不使用代理： NSURLSession的dataTaskWithRequest和dataTaskWithURL
    • 使用代理： NSURLSession的 dataTask:didReceiveData和 task:didCompleteWithError,前者收到数据时通知，后者收到全部数据时通知
    • 下载文件：
      • 创建配置，后台下载使用后台会话配置
      • 在会话中创建或者恢复下载任务
      • 用代理处理任务： downloadTask:didFinishDownloadintToURL task:didCompleteWithError
    • POST：使用 NSMutableURLRequest
      • 小的内存数据，直接编码到URL数据中
      • 上传硬盘文件，setHTTPBodyStream到文件的NSInputStream，然后把结果数据作为HTTP请求的body content
      • 上传大块数据，用CFStreamCreateBoundPair创建乒乓文件流，同样用setHTTPBodyStream到其中一个流进行上传，同时向另一个流写入
      • 设置HTTP头 setValue:forHTTPHeaderField
      • 获取上传状态 connection:didSendBodyData:totalBytesWritten:totalBytesExpectedToWrite
    • 身份验证：URLSession:task:didReceiveChallenge:completionHandler
  • 使用Core Foundation框架（略）
  • 其它：
    • NSJSONSerializaion用于本地数据和JSON的转换
    • NSXMLParser XML的SAX解析API
    • NSXMLDocument XML的DOM解析API

网络安全

  • 常见网络攻击：
    嗅探snooping
    中间人man-in-the-middle:嗅探和钓鱼spoofing and phishing，篡改tampering，会话劫持session hijecking
    注入式攻击injection attacks
    缓存溢出,值溢出buffer overflows， numeric overflows
  • 在IOS中使用 https前缀访问网站

坑

  • 不要在主队列中使用同步网络调用，要使用异步网络调用
