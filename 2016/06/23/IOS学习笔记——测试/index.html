IOS学习笔记——测试

  1. 调试
  2. 功能测试
  3. 性能测试
  4. UI测试

IOS学习笔记，介绍Xcode下如何进行调试和测试

调试

  • 在Debug Area下按Debug View Hierarchy按钮，可以以3D的形式查看View层级
  • 按Quick Look按钮可以以图形的形式快速查看变量内容
  • 在Schema-Diagnostics-Runtime Sanitisation中勾选Enable Address Sanitizer可以查看内存崩溃的具体地址

功能测试

  • XCTest是xcode的测试框架
  • 测试用例结构：TestBundle->TestClass->TestMethod
  • 测试类的内容：setUp, tearDown testExample tesPerformanceExample
  • 对异步操作进行测试，用XCTestExpectation。在异步操作的completion中设定Expectation满足，在异步操作外waitForExpectationWithTimeout
  • 对Swift进行测试，需要在构建时选Enable testability选项，同时在测试用例中引用Swift类时写@testable import XXXX
  • 断言：
    XCTFail
    XCTAssertEqualObjects/Not
    XCTAssertEqual/Not
    XCTAssertEqualWithAccuracy/Not
    XCTAssertGreaterThan/Less
    XCTAssertGreaterOrEqualThan/Less
    XCTAssertNil/Not
    XCTAssertTrue/False
    XCTAssertThrows/ThrowsSpecific/NoThrow/NoThrowSpecific
  • 可以在测试中设定测试失败断点
  • 发生异常时测试会停止，可以禁止异常断点，保证测试继续进行？
  • 在测试的scema中可以选择是否检查代码覆盖率，在详细报告中还可以看到每句被执行了多少次

    性能测试

  • XCTest的measureBlock
  • 在性能测试结果窗口中，可以修改测试基线和标准差

    UI测试

  • UI测试包括：UI记录，UI测试。UI测试基于XCTest和Accessibility框架
  • API:XCUIApplication XCUIElement XCUIElementQuery
  • UI记录：
    • 在测试导航页面，创建一个UI测试目标
    • 在创建的模板文件中，用光标指向测试函数
    • 开始记录UI操作
    • 停止UI操作
    • 添加断言
  • UI测试：
    • 查询元素XCUIElementQuery
    • 了解元素期望的行为
    • 对元素进行交互
    • 检查响应结果与期望行为是否一致
  • setUp:
    continueAfterFailure = false
    XCUIApplication().launch()
  • 测试用例：
    let app = XCUIApplication()
    sleep(xx) 在这里最好等一下app完全初始化好，否则第一个操作可能不起作用
    录制的一些操作
    XCAssertEqual(app.staticTexts["xxxx"].label, "xxx"查询UI元素，这里的UI元素最好指定了accessibility的identifier, 元素不能转型成本来的UI类型，只能通过XCUIElementAttribute来查看属性
