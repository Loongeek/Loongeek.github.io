Git简介

版本控制系统(Version Control System, VCS)

  • 本地版本控制：无法远程共享
  • 集中式版本控制：CVS, Subversion. 服务器必须一直正常工作，客户端也必须连接到服务器，否则难以进行版本控制。服务器如果坏掉，有可能丢失所有版本记录
  • 分布式版本控制：每个客户端拷贝版本库而不是版本快照，支持多客户端/多服务器的互相共享和备份

    Git特点

  • Git的每个Version是文件快照，是到每个文件的链接集合，而不是文件补丁的集合

  • Git大部分操作可以在本地进行
  • Git集成了文件完整性检查，采用SHA-1校验和的方式来保证，Git中的文件以哈希值而不是文件名区分
  • Git通常只会增加数据，很难undo或者删除数据
  • Git有三个状态：
    - `committed`已提交，说明数据已经安全地保存在本地数据库中(保存到Git仓库中)
- `modified`已修改，说明数据已经修改，但还未存在本地数据库中(数据存在于工作区中)
- `staged`已暂存，标记已修改文件的当前版本，以便下次提交(数据存在于暂存区中)
![三个状态之间的转换图](https://git-scm.com/book/en/v2/book/01-introduction/images/areas.png)
  • Git的一般工作流程：
    - 在工作目录中修改文件(文件状态变成`modified`)
- 将文件暂存，在暂存区中添加快照(文件状态变成`staged`)
- 提交文件到Git仓库(文件状态变成`committed`)

Git配置

  • 配置工具git config
  • 配置变量：
    `/etc/gitconfig` 系统级配置, `git config --system`
`~/.gitconfig`或`~/.config/git/config` 当前用户配置， `git config --global`
`.git/config` 当前仓库配置，`git config`
范围小的配置覆盖范围大的配置
  • 安装完Git后首先配置用户名和邮件地址
    git config --global user.name "Loongeek"
    git config --global user.email loongeek@mail.com
    再配置默认文本编辑器
    git config --global core.editor vim
    可以检查配置
    git config --list或者git config <KEY>

Git基本操作

获取Git仓库

  • 本地：
    `git init` 建立仓库
`git add XXX` 添加文件
`git add LICENSE`
`git commit -m 'init'` 提交，标签为'init'
  • 远程：
    `git clone https://github.com/XXXX [LOCAL DIR]` 从远程克隆仓库到本地目录

    控制文件状态

  • 检查文件状态：git status或git status -s(紧凑输出)
  • 跟踪新文件：git add XXX
  • 暂存已修改文件：git add XXX
  • 无需版本控制的文件，在.gitignore中指定。针对不同项目的.gitignore
  • 查看具体修改：git diff查看未暂存文件的修改
    `git diff --staged`查看已暂存文件的修改
  • 提交更新 git commit或者git commit -m <COMMENTARY>
    注意：提交的是暂存区的内容，未暂存的内容不会提交
  • 可以跳过暂存过程 git commit -a相当于add后直接commit
    • 移动文件 git mv <FROM> <TO>
    • 删除文件 git rm <FILE>将文件从暂存区和工作目录中删除，下次提交后就将文件从版本管理中删除
      `git rm -f <FILE>`强行删除，针对文件已修改且放入暂存区的情况
`git rm --cached <FILE>`从暂存区中删除文件，但是不从工作目录中删除文件

      查看历史：

      git log 列出所有日志
      git log -p -<COUNT> 显示每次提交的差异，显示最近COUNT次提交
      git log --stat 显示每次提交的统计信息
      git log --pretty=<oneline|short|full|fuller> 格式化输出
      还可以使用输出过滤器等等

      撤消

    • 撤消上次提交：git commit --amend
      • 取消暂存：git reset HEAD <FILE>
      • 取消修改：git checkout -- <FILE>

        远程仓库

    • 查看远程仓库：git remote
      • 添加远程仓库：git remote add <SHORTNAME> <URL>
      • 删除远程仓库：git remote rm <NAME>
      • 抓取本地没有的数据：git fetch <REMOTE-NAME>,不会自动合并
      • 拉取远程数据：git pull <REMOTE-NAME>，会自动合并
      • 推送到远程：git push origin master
      • 查看远程仓库内容：git remote show
      • 重命名远程仓库：git remote rename <OLD> <NEW>

        标签

    • 显示：git tag git tag -l '<PATTERN>'
    • 附注标签，加入了更多信息 git tag -a <TAG> -m <MESSAGE>
      显示git show <TAG>
    • 轻量标签，git tag <TAG>
    • 补打标签， git tag -a <TAG> <HASHCODE>
    • 标签必须显式的push到远程 git push origin <TAG>
      git push origin --tags一次推送所有标签

Git分支

Git提交对象

首次提交生成文件快照的blob(binary large object，二进制大对象)，树对象和提交对象

后续提交的父对象是上次提交

分支就是在某次提交对象上创建指针

通常采用多分支并行前进的流水线型分支s

分支操作

  • 创建分支：git branch <NAME>，仅创建分支，不切换
  • 查看分支：git log --decorate
    `git log --decorate --graph --all` 图形化显示
  • 切换分支：git checkout <NAME> 让HEAD指向该分支
    `git checkout -b <NAME>` 创建并切换分支
  • 合并分支：git checkout <MAIN-BRANCH>
    `git merge <SUB-BRANCH>` 将分支合并到主分支
  • 删除分支：git branch -d <NAME>
    `git branch -D <NAME>` 强行删除未合并的分支
  • 冲突处理：处理完成后，对每个文件使用git add暂存，最后再git commit提交
  • 查看分支 git branch
    `git branch -v`
`git branch --merged`
`git branch --no-merged`

    远程分支

  • 获取远程分支 git fetch origin+git merge origin/<BRANCH> 或者 git pull origin/<BRANCH>
  • 推送到远程 git push origin <BRANCH>
  • 删除远程分支 git push origin --delete <BRANCH>

变基

  • 变基操作：git checkout <SUB-BRANCH>
    `git rebase <MAIN-BRANCH>` 将分支变基到主分支上
`git checkout <MAIN-BRANCH>`
`git merge <SUB-BRANCH>` 将主分支指向最新
    变基与合并的结果一样，但是变基将日志流变成一条单线，提交到服务器会比较整洁
  • 高阶变基：git rebase master client 将client变基到master上
    git rebase --onto master client2 client1将client1的改变变基到master上，但忽略client2独有的改变

注意：千万不要对已经提交到远程的内容执行变基操作，否则会导致日志流混乱而且出现重复

GitHub

Git工作流程

  • 类似SVN的集中式工作流
  • 集成管理式(Hub式管理)

GitHub

  • Fork：在自己的空间建立一个独立副本。如果想参与某个项目但是没有推送权限可以用fork,并可以通过创建pull request推送
  • 流程：clone->checkout new branch->commit->push origin new branch->pull request->discuss->merge or close
  • 合并讨论中可以使用GitHub风格的Markdown
  • 合并讨论中可以@其它人

    Git工具

    储藏(stash)

  • 储藏：git stash，将当前工作区的修改wyadgc栈上
  • 显示所有储藏：git stash list
  • 取回储藏：git stash apply或git stash apply <NAME>
  • 取回储藏并暂存当时已暂存的文件：git stash apply --index
  • 取回储藏时创建新分支(避免合并冲突)：git stash branch <NAME>
  • 从栈上删除储藏：git stash drop或git stash drop <NAME>
  • 只储藏未暂存的文件：git stash --keep-index
  • 储藏时包括未跟踪文件：git stash -u

    清理

  • 清理未跟踪且未被忽略的文件： git clean -d
    被清理的文件有可能无法恢复，更安全的方法：git stash --all缓存到栈中
  • 强制清理：git clean -f -d
    只显示要清理的文件，但不真正清理：git clean -d -n
    同时清理被忽略的文件：git clean -d -x
    交互式清理：git clean -d -i

    重置(reset)

  • 软重置：git reset --soft HEAD~
  • 普通重置：git reset --mixed HEAD~或git reset HEAD~
  • 硬重置：git reset --hard HEAD~
    注意：硬重置后，已提交的内容可以用reflog找回来，但是工作目录中的未提交内容就完全无法恢复了
  • reset --hard会移动HEAD及HEAD下的分支，checkout只会移动HEAD

    撤消合并

  • 修复引用法

    类似变基，会重写提交历史。如果有人已经获取了合并后的内容，会产生混乱
  • 还原提交法
    git revert -m 1 HEAD 数字表示还原到哪个父对象

    git revert ^M+git merge topic 要合并时需要先还原还原操作
